# DevOps Week2

## Описание

В этом репозитории собраны Ansible-роли для автоматической настройки Debian-системы.
Проект демонстрирует работу с Ansible, локальный запуск через inventory, установку базового программного обеспечения, настройку пользователей, развёртывание Docker и установку Node Exporter.

Все действия выполняются единым плейбуком `site.yml`, который применяет все роли к локальному хосту.

## Структура репозитория

```text
roles/
  base/
    tasks/main.yml
    handlers/main.yml
  docker/
    tasks/main.yml
  node_exporter/
    tasks/main.yml
inventory.ini
site.yml
README.md
```

## Требования

Для запуска проекта нужны:

* Debian
* установленный Ansible
* доступ к интернету
* `sudo` без пароля для текущего пользователя
* локальный запуск Ansible через inventory

Пример `inventory.ini`:

```ini
[local]
localhost ansible_connection=local ansible_python_interpreter=/usr/bin/python3
```

## Как запустить

Перед запуском нужно перейти в корень репозитория.

Команда запуска:

```bash
ansible-playbook -i inventory.ini site.yml
```

Эта команда применяет все роли к локальной системе и выполняет полную настройку автоматически.

## Что делает каждая роль

### `roles/base`

Роль отвечает за базовую настройку системы.

Она выполняет следующие действия:

* устанавливает необходимые пакеты
* создаёт пользователей `dev` и `monitoring`
* настраивает файл приветствия `motd`
* открывает порт `8080` в firewall
* включает firewall
* настраивает Nginx для работы на порту `8080`
* создаёт cron-задачу для пользователя `monitoring`
* подготавливает файл `/var/log/heartbeat.log`

### `roles/docker`

Роль отвечает за установку Docker по официальной схеме через `apt`.

Она выполняет следующие действия:

* устанавливает зависимости для Docker
* добавляет GPG-ключ Docker
* добавляет официальный репозиторий Docker
* устанавливает `docker-ce`, `docker-ce-cli`, `containerd.io`
* создаёт группу `docker`
* добавляет пользователя `dev` в группу `docker`
* запускает и включает службу Docker
* запускает тестовый контейнер `hello-world`

### `roles/node_exporter`

Роль отвечает за установку и запуск Node Exporter.

Она выполняет следующие действия:

* создаёт системного пользователя `node_exporter`
* скачивает архив Node Exporter с GitHub
* распаковывает его в `/opt/node_exporter`
* создаёт символическую ссылку `/usr/local/bin/node_exporter`
* создаёт systemd-юнит
* запускает и включает службу
* проверяет доступность метрик на `localhost:9100`

## Проверка после запуска

После выполнения `site.yml` можно проверить результат вручную.

Проверка Nginx:

```bash
curl http://localhost:8080
```

Проверка Node Exporter:

```bash
curl http://localhost:9100/metrics
```

Проверка статуса служб:

```bash
systemctl status nginx
systemctl status docker
systemctl status node_exporter
```

Проверка пользователей:

```bash
id dev
id monitoring
```

Проверка Docker:

```bash
docker run hello-world
```

## Примечания

Если после добавления пользователя `dev` в группу `docker` вход уже был открыт, может потребоваться перелогиниться, чтобы новая группа применилась.

Если Node Exporter не отвечает, нужно проверить статус службы и убедиться, что порт `9100` свободен.

Если `ufw` не установлен, его нужно добавить через `apt`, иначе роль `base` не сможет открыть порт `8080`.

## Результат

После успешного запуска проекта система получает готовую базовую конфигурацию, Docker и Node Exporter, управляемые через Ansible-роли.
