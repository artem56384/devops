# Сигналы в Linux

## Что это

Сигналы — это механизм взаимодействия между процессами.

## Основные сигналы

### SIGTERM (15)

Используется для корректного завершения процесса.

Особенности:
- процесс может перехватить сигнал
- может выполнить очистку ресурсов
- может сохранить данные перед завершением

Это основной способ остановки процессов.

---

### SIGKILL (9)

Принудительное завершение процесса.

Особенности:
- не может быть перехвачен или обработан
- процесс завершается немедленно
- не выполняется очистка ресурсов
- возможна потеря данных

Используется, если процесс не реагирует на SIGTERM.

---

### Разница SIGTERM и SIGKILL

SIGTERM:
- корректное завершение
- безопасно для данных

SIGKILL:
- жёсткое завершение
- используется как крайняя мера

---

## Команды

kill <pid>  
kill -15 <pid>  
kill -9 <pid>  
pkill <name>  

## Вывод

SIGTERM используется для корректного завершения,  
SIGKILL — для принудительного завершения процесса.
