# Сигналы в Linux

## Что такое сигналы

Сигналы в Linux — это механизм взаимодействия между процессами и системой. Сигнал представляет собой уведомление процессу о наступлении какого либо события.

С помощью сигналов можно завершать процессы, приостанавливать их, продолжать выполнение и сообщать об ошибках.

## Основные типы сигналов

Сигналы можно условно разделить на несколько групп:

1. Сигналы завершения процесса
2. Сигналы остановки и продолжения
3. Сигналы, связанные с ошибками
4. Пользовательские сигналы

## Основные сигналы

### SIGINT

Номер сигнала: `2`

Отправляется при нажатии `Ctrl + C` в терминале. Обычно используется для прерывания процесса.

### SIGTERM

Номер сигнала: `15`

Основной сигнал для корректного завершения процесса. Процесс может обработать его и завершиться правильно.

### SIGKILL

Номер сигнала: `9`

Принудительное завершение процесса. Процесс не может перехватить или игнорировать этот сигнал.

### SIGHUP

Номер сигнала: `1`

Часто используется для перезагрузки конфигурации процесса без полной остановки.

### SIGSTOP

Останавливает выполнение процесса. Процесс не может игнорировать этот сигнал.

### SIGCONT

Продолжает выполнение ранее остановленного процесса.

### SIGUSR1 и SIGUSR2

Пользовательские сигналы, которые программы могут использовать для собственных целей.
